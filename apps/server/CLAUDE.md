# apps/server — Backend (learning zone)

This is the **primary learning surface** of the monorepo. The user is here to learn backend development hands-on. Treat this folder differently from the rest of the project.

## Collaboration rules in this folder

- **The user writes the code by default.** When asked a question, prefer to explain, point at examples, and let the user implement. Resist the urge to immediately produce a full diff.
- **Explain unfamiliar Node/Fastify/Drizzle/Postgres/Zod concepts** as they come up. Assume the user is competent in TypeScript and frontend, but is newer to backend specifics: request lifecycle, middleware/hooks, ORMs, migrations, transactions, auth flows, etc.
- **Do not skip ahead.** The course progresses step by step (see root `CLAUDE.md`). Don't pre-introduce auth, roles, transactions, etc. before they're the topic.

## Stack

- **Fastify 5** + `fastify-type-provider-zod` for typed request/response schemas
- **Drizzle ORM** + `drizzle-kit` for migrations against PostgreSQL
- **Zod** for input/output validation
- **Swagger** (`@fastify/swagger` + swagger-ui) for autogenerated API docs and FE typings (basic-auth gated: `admin` / `supersecure123`)
- **PostgreSQL** via `docker-compose.local.yml`

## Layout

```
src/
├── api/
│   ├── api.ts              # server bootstrap
│   ├── errors/             # custom error classes (thrown by controllers, mapped to HTTP)
│   ├── hooks/              # Fastify lifecycle hooks
│   ├── plugins/            # Fastify plugins (cors, helmet, swagger, drizzle, etc.)
│   └── routes/             # route definitions; uses autoload + autohooks
├── controllers/            # business logic per entity (post, comment, entity)
├── repos/                  # DB access via Drizzle (one repo per entity)
├── services/               # cross-cutting infra (drizzle client, env, uuid)
└── types/                  # shared TS types + Zod-derived schemas
```

Routing convention: file/folder structure under `src/api/routes` mirrors URL structure (autoload). Each route file declares Zod schemas and delegates to a controller; controllers call repos.


Swagger UI: `http://localhost:3000/api/documentation` (basic auth `admin` / `supersecure123`).

## Conventions to preserve

- **Schemas live with routes** (under `src/api/routes/.../schemas/` patterns) and are referenced from route definitions for both validation and OpenAPI generation.
- **Controllers throw typed errors** from `src/api/errors`; the error handler maps them to HTTP responses. Don't `reply.code(...).send(...)` from controllers.
- **Repos return plain data** and own all Drizzle calls. Controllers should not import `drizzle-orm` directly.
- **Env access** goes through `services/env`, not `process.env` scattered around.
- **Imports** use `tsconfig-paths` aliases — keep them; don't rewrite to relative paths.
